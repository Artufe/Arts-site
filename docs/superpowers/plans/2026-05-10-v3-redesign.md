# v3 Redesign — Brutalist/Terminal (v3.1 accent)

> **Artifact reference:** `arthur-site-v3.1.html` at project root — a standalone HTML prototype of the full redesign. Consult it for visual intent, layout posture, and animation patterns. It is **not** the implementation — the Next.js app architecture, command palette, snake game, and route structure in the existing codebase must be preserved and adapted, not replaced.

**Goal:** Apply the v3.1 brutalist/terminal design language to the existing Next.js 15 Arts-site, muting the accent from the `arthur-site-v3.html` red to a warm rust (`oklch(50% 0.14 22)` light / `oklch(55% 0.16 25)` dark). Keep all existing content, routes, and interactive features (command palette, snake game, hero shader, theme toggle, contact form, MDX case studies). Replace the visual layer — CSS tokens, component layouts, typography, animation — while reusing components where the architecture already fits.

**Design language (brutalist/terminal):**
- Monospace body (`ui-monospace, 'IBM Plex Mono', 'JetBrains Mono', Menlo`) across the entire site
- Serif display (`'Times New Roman', 'Iowan Old Style', Georgia`) for headlines — large, tight letter-spacing
- 2px borders everywhere, zero border-radius everywhere (already enforced)
- Scan-line CRT overlay as a fixed pseudo-element or grain replacement
- Command bar on home page (command palette already exists — adapt its visual, keep its behavior)
- Typewriter effect + pulse animations on key elements
- Timeline dot with pulse ring
- Dot-grid background texture (replaces or augments existing grain)
- Asymmetric layouts: `grid-template-columns: 3fr 1fr` for content + sidebar

**What stays (minimal changes needed):**
- Command palette (`components/command-palette.tsx`, `lib/commands.ts`, `lib/fuzzy.ts`) — adapt visual only
- Snake game (`components/snake/*`, `lib/snake-bus.ts`, `app/snake/page.tsx`) — adapt visual only
- Hero shader (`components/hero-shader.tsx`, `public/hero-shader.js`) — keep as-is, route gating already works
- Theme provider (`components/theme-provider.tsx`) — keep as-is
- Theme toggle (`components/theme-toggle.tsx`) — keep as-is
- Contact form (`components/contact-form.tsx`) — keep as-is
- MDX infrastructure (`mdx-components.tsx`, `components/mdx/*`) — adapt typography tokens
- Footer (`components/footer.tsx`) — adapt visual only
- Content modules (`content/site.ts`, `content/cv.ts`, `content/about.ts`) — keep as-is
- Routes (`app/*/page.tsx`) — adapt imports, keep structure
- Build & deploy config — no changes needed

## Task 1: Rewrite CSS tokens (`app/globals.css`)

**Files:**
- Modify: `app/globals.css`

Replace the entire file with the v3.1 brutalist token system. Keep Tailwind v4 `@import 'tailwindcss'` + `@theme` block, but update every color, font, and surface token. Key token changes:

- `--bg`: `oklch(96% 0.004 100)` light → `oklch(10% 0.008 100)` dark
- `--fg`: `oklch(15% 0.02 100)` light → `oklch(90% 0.004 100)` dark
- `--accent`: `oklch(50% 0.14 22)` light → `oklch(55% 0.16 25)` dark (muted rust)
- `--border`: same as `--fg` (2px everywhere, not a muted grey)
- `--rule`: replaced by `--border`
- `--font-display`: `'Times New Roman', 'Iowan Old Style', Georgia, serif`
- `--font-body`: `ui-monospace, 'IBM Plex Mono', 'JetBrains Mono', Menlo, monospace`
- Remove `--card`, `--panel-shadow` (no shadows in brutalist)
- Remove `--accent-glow`, `--live-glow` (replaced by solid accent)
- Add `--scan` for scan-line overlay color

Key rules to add:
- `body` uses `--font-body` (monospace everywhere, including body text)
- `h1, h2, h3, h4, h5, h6` use `--font-display`
- All borders become 2px solid `var(--border)` — no 1px subtle borders
- Keep `* { border-radius: 0 !important; }`
- Add style for fixed scan-line overlay pseudo-element
- Add `@keyframes blink`, `@keyframes pulse-ring`, `@keyframes count-up`
- Add `@keyframes appear` for scroll-reveal
- Update `prefers-reduced-motion` block

## Task 4: Scan-line overlay + dot grid background

**Files:**
- Modify: `components/grain.tsx` (rename conceptually, or create `components/scan-line.tsx`)

The current `components/grain.tsx` renders an SVG fractal noise overlay at 1.5% opacity. The v3.1 design replaces this with:

1. **Fixed scan-line overlay** — repeating horizontal lines, 2px transparent, 4px cycle. Implemented as a CSS `body::after` pseudo-element with `background: repeating-linear-gradient` (same technique as the v3.1 prototype). Pointer-events none, z-index 9999.
2. **Dot-grid background** — subtle dot pattern on the body background. Implemented via a CSS `background-image` with `radial-gradient` dots at 24px spacing, ~6% opacity.

Both can live in `app/globals.css` as body pseudo-elements, eliminating the need for a separate `components/grain.tsx` component entirely. However, for consistency with existing architecture, keep the component file and render the scan-line + dot-grid from a single `components/scan-line.tsx` component mounted in layout.

## Testing Philosophy

Same as existing project:
1. **TypeScript strict + `tsc --noEmit`** — catch most bugs
2. **`npm run build`** — primary verification (routes, imports, MDX, static export)
3. **Targeted unit tests** — `npm test` for existing tests (contact-form, theme-toggle, button, utils, snake-engine)
4. **Manual smoke** — each page rendered once in browser, dark/light toggled
5. **E2E** — `npm run test:e2e` with dev server running (Playwright smoke tests)
